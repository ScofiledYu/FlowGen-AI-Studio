# FlowGen AI Studio — 部署说明

本文说明如何在 **局域网** 环境部署当前前端与 Node 代理服务，以及如何安装 **SQL Server Express**（供后续接入登录、项目与画布持久化等后端时使用）。更细的核对项见 [`部署检查清单.md`](部署检查清单.md)，拷贝文件列表见 [`服务器部署文件清单.md`](服务器部署文件清单.md)。

---

## 1. 架构简述

| 组件 | 说明 |
|------|------|
| 静态前端 | Vite 构建产物目录 `dist/`，由 `server.js` 托管 |
| Node 服务 | Express（`server.js`），提供静态资源、`/api` 代理、`/flowgen-api` 多用户 JSON API、`/proxy-file` 等 |
| 多用户数据（当前实现） | 默认目录 **`data/flowgen/`**（`store.json` + `uploads/`）；生产推荐 **MySQL 5.7** 快照 + 同上 `uploads/`（见 [`docs/mysql-deployment.md`](docs/mysql-deployment.md)） |
| 数据库（规划） | **SQL Server Express**（历史规划）；**当前生产以 MySQL 为准** |

默认监听：**端口 `3000`**（环境变量 `PORT` 可覆盖）。生产环境建议通过 `0.0.0.0` 监听以便局域网其它机器访问（以 [`server.js`](server.js) 实际配置为准）。

## 2. 环境要求

### 2.1 运行 Node 应用的服务器 / 开发机

- **Windows 10**（开发机）或 **Windows Server**（局域网服务器）
- **Node.js**：建议 **LTS**（与 [`package.json`](package.json) 锁定的依赖兼容即可）
- 能访问外网时，需满足 AI 相关接口可达（详见 [`部署检查清单.md`](部署检查清单.md) 中外包依赖一节）

### 2.2 数据库服务器（可选，建议提前装好）

- 若生产环境为 **Windows Server 2012**：请使用 **SQL Server 2016 SP2 Express**（见下文），**不要**在 2012 上安装 SQL Server 2019/2022（官方安装包通常不支持该操作系统）。
- **Windows 10 开发机**若希望与生产库行为一致，建议安装 **与服务器相同主版本** 的 Express（例如同为 2016 SP2）。

## 3. SQL Server Express 安装（官方安装包）

以下内容仅为 **下载与选型指引**，请在浏览器中打开微软官方页面下载；勿从未知第三方站点获取安装包。

### 3.1 下载地址

| 场景 | 安装包 | Microsoft 下载中心 |
|------|--------|---------------------|
| **Windows Server 2012**（推荐用于该机） | SQL Server **2016 SP2** Express | https://www.microsoft.com/zh-cn/download/details.aspx?id=56840 |
| **Windows 10**（与上面生产环境保持一致） | 同上 **2016 SP2** Express | 同上 |
| **仅 Win10**、且确认生产环境为新系统（2016+）时可选 | SQL Server **2019** Express | https://www.microsoft.com/zh-cn/download/details.aspx?id=101064 |

下载后一般为小型引导程序（如 `SQLServer2016-SSEI-Expr.exe`），按向导选择「数据库引擎」等组件；实例名、身份验证方式（Windows 身份验证 / 混合模式）请在团队内约定一致，便于连接字符串与备份策略统一。

### 3.2 管理工具（推荐）

- **SQL Server Management Studio (SSMS)**：用于建库、执行脚本、备份还原。  
  文档入口：<https://learn.microsoft.com/zh-cn/ssms/download-sql-server-management-studio-ssms>

安装 SSMS **不会**替代数据库引擎本身；引擎与 SSMS 可分别安装在服务器或开发机上。

## 4. 构建与部署 Node + 前端

### 4.1 在开发机生成构建产物

在项目根目录执行：

```powershell
cd /d D:\aaa\flowgen-ai-studio
npm install
npm run build
```

成功后生成 `dist/` 目录。

### 4.2 拷贝到服务器（最小集）

典型需要包含：

- `server.js`、`promptPlaceholders.mjs`
- 目录 **`server/flowgen/`**（多用户 API）
- `package.json`、`package-lock.json`
- 完整目录 `dist/`

完整列表与「是否拷贝 node_modules」两种方式见 [`服务器部署文件清单.md`](服务器部署文件清单.md)。

### 4.3 在服务器安装依赖并启动

```powershell
cd <解压后的项目目录>
npm install --production
npm start
```

或将 `npm start` 注册为 Windows 服务 / 计划任务，保证机器重启后自动拉起。

**多用户 / 项目 / 资产库（`/flowgen-api`）**

- 数据默认在应用工作目录下的 **`data/flowgen/`**（可用环境变量 **`FLOWGEN_DATA_DIR`** 改到其它路径以便备份）。
- **种子超级管理员**（勿把真实密码提交到 Git）：`FLOWGEN_BOOTSTRAP_ADMIN_USER`、`FLOWGEN_BOOTSTRAP_ADMIN_PASSWORD`（均未设置时默认为 `admin` / `admin`，首次登录会要求修改密码）。
- **JWT**：生产环境务必设置 **`FLOWGEN_JWT_SECRET`**；可选 **`FLOWGEN_JWT_EXPIRES`**（如 `7d`）。
- **仅开发**：`npm run dev:full` 会并行启动 Vite 与端口 **3001** 的 FlowGen API；浏览器仍访问 `http://localhost:3000`，`/flowgen-api` 由 Vite 转发到 3001。
- **不登录单机使用**：打开 **`#/legacy`** 仍走原先仅 localStorage 的流程。

### 4.4 防火墙与局域网访问

- 放行 **入站 TCP**：应用端口（默认 **3000**）。
- 客户端浏览器访问：`http://<服务器局域网IP>:3000`。

## 5. 配置与运维提示

- **端口**：若需改为 80/443，可调整 `PORT` 或使用 IIS/Nginx 反向代理到 Node（实施方式依运维规范而定）。
- **HTTPS**：对外或跨域场景建议在反向代理层配置 TLS；本文不展开证书细节。
- **日志与崩溃**：结合 [`部署检查清单.md`](部署检查清单.md) 中的运行与环境检查项定期核对。
- **数据库备份**：接入 SQL Server 后，应对业务库做定期备份（完整备份 + 日志策略按数据重要性制定）。
