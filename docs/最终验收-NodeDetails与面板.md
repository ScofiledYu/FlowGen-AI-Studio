# 最终验收：Node Details + 属性面板 + @ 引用

本地地址：**http://localhost:3001**（改代码后需 `npm run build` 并强刷 **Ctrl+Shift+R**）

自动化（不调用生成 API）：

```bash
npm run test:final
```

通过即表示：全模型面板槽位、@ 定位、上下游 Node Details 逻辑已与代码一致。

---

## 一、通用检查（每个模型都做）

| 检查项 | 上游运行节点 | 生成节点（输出） |
|--------|-------------|------------------|
| **Model** | 与侧栏所选模型一致 | 与本次运行快照 `generationParams.model` 一致，**不是**侧栏下一步默认模型 |
| **Prompt** | 当前 **tab** 对应文案（Omni 用 `klingOmniMultiPrompt` 等） | 本次运行快照中的 `prompt` |
| **参考图** | 当前 tab 面板槽位 + `@主图` 时的主预览；**无**重复 blob/三合一 | 仅本次 API 实际提交的 `referenceImages` 顺序与张数 |
| **属性面板** | 运行后面板槽位 **不减少、不乱序**；未 `@` 的格仍在原索引 | — |

## 二、分模型 / 分 Tab 点验清单

### 1. Nano Banana 2.0

- [ ] 侧栏：主图格 + 参考图格；`@主图` + `@图片1`
- [ ] 运行后：面板 2 槽仍在；Node Details 上游 2 张；输出节点 Details 2 张

### 2. image 2

- [ ] 三槽主图+参考；`@图片3` + `@图片1`（逆序 @）
- [ ] 运行后仍为 3 槽；Details 仅含已 @ 的 2 张

### 3. 可灵 2.5 Turbo

- [ ] 首尾帧 + 可选参考图；`@首帧图` / `@尾帧图`
- [ ] 上游 Details 含首帧 URL；输出节点参数为 quality/duration/aspectRatio 快照

### 4. vidu 2.0

- [ ] 首尾帧；`@首帧图`
- [ ] Details 参数含 viduDuration / viduClarity

### 5. seedance1.5-pro（图生 tab）

- [ ] 首尾帧模式；`@首帧图` `@尾帧图`

### 6. seedance2.0（高质量 / 急速）

- [ ] **参考生**：`@主图` + `@图片n`，主图不进 reference 槽位挤占
- [ ] **图生**：首尾帧 `@首帧图`
- [ ] **文生**：无参考图时 Details 参考区为空或仅文案相关

### 7. 即梦 3.0 Pro

- [ ] 图生：仅首帧槽（无多图参考区）；`@首帧图` 或 `@主图`
- [ ] 上游 Details 与面板一致

### 8. 可灵 3.0 Omni（四个 tab 各测一次）

| Tab | 面板 | @ 示例 | 注意 |
|-----|------|--------|------|
| **多图参考** | **主图格** + 多图参考格 | `@主图` `@图片1` | 主图不可丢；无 `blob:…mov` 重复 |
| **指令变换** | 图 + 视频槽 | `@图片1` `@视频1` | |
| **视频参考** | 图 + 视频 | `@图片1` | |
| **首尾帧** | 首帧 / 尾帧 | `@首帧图` | |

- [ ] 每个 tab 切换后创意描述变为该 tab 文案
- [ ] 生成节点 Details：参考图张数 = 本次 @ 提交张数（通常 1–2，非 3 张重复）

## 三、推荐验收流程（约 15 分钟）

1. 打开项目 `flowgen-project-2026-04-24.json`（或当前在修的画布）。
2. 任选一条 **可灵 3.0 Omni 多图** 链路：看上游节点属性面板是否有 **主图**；打开上游 / 输出 **Node Details** 对照上表。
3. 对 **Nano、image2、Seedance 参考生** 各跑 1 次（可不真等 API 完成，看提交前面板是否已稳定）。
4. 强刷后重复步骤 2，确认无旧 `dist` 缓存。

---

## 四、已知说明

- **旧输出节点**若无 `generationParams`，需 **重新运行** 后 Details 才有完整参考图快照。
- 仅拖入未 `@` 的素材 **不会** 进入 API，但应仍在面板原槽位。
