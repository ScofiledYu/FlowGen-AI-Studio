# FlowGen 服务器部署清单（日常维护）

首次安装（步骤+命令合一）：[Windows-Server-2012R2-离线部署说明.md · 第一部分](docs/Windows-Server-2012R2-离线部署说明.md#第一部分首次安装)

程序 `D:\apps\flowgen-ai-studio\` · 端口 3001 · MySQL 服务 MySQL57

**命令怎么复制** — 【整段复制】一次粘贴 · 【逐行执行】一行一行 · 【手动】资源管理器或 Ctrl+C

---

## 每次更新（步骤 1～7）

**步骤-1** · 有网电脑

- 只改界面 → 拷 `dist\`：

【整段复制】

    cd D:\aaa\flowgen-ai-studio
    npm run build

- 改后端、依赖未变 → 拷 `server.js`、`promptPlaceholders.mjs`、`server\`、`scripts\`
- 依赖变了或全拷 → 拷 8 项 + 执行：

【整段复制】

    cd D:\aaa\flowgen-ai-studio
    npm ci
    npm run build

- 不要盖 `.env.local`、`data\`；只拷 dist/4 项时不要盖 `node_modules\`
- **本次及以后全量更新**须含：`dist\`、`server.js`、`promptPlaceholders.mjs`、`server\`（含 `workspacePayloadCodec.mjs`）、运行时 `utils\`（含 `aitopChatModels.ts` 等）、`package.json`；漏拷会导致 MySQL/Chat/MiniMap 与开发机不一致
- 服务器 `.env.local` **必须**有 `FLOWGEN_JWT_SECRET`（与开发机一致要求；勿用仓库默认值）。可对照根目录 `.env.example`

---

**步骤-2** · 服务器 · 停 FlowGen

【手动】npm start 窗口 Ctrl+C

---

**步骤-3** · 服务器 · 备份（仅拷 8 项）

【手动】备份 `.env.local` 到 U 盘

---

**步骤-4** · U 盘 · 覆盖

【手动】覆盖到 `D:\apps\flowgen-ai-studio\`，不要盖 `data\`

---

**步骤-5** · 服务器 · 还原（仅拷 8 项）

【手动】`.env.local` 拷回程序目录

---

**步骤-6** · 服务器 · 启动

【逐行执行】`npm start` 单独执行：

    net start MySQL57
    cd D:\apps\flowgen-ai-studio
    $env:PORT = "3001"
    npm start

---

**步骤-7** · 浏览器

【手动】Ctrl+F5

---

## 开关机

**开机** — 同步骤-6 【逐行执行】

    net start MySQL57
    cd D:\apps\flowgen-ai-studio
    $env:PORT = "3001"
    npm start

**关机** — 【手动】Ctrl+C，再 【逐行执行】`net stop MySQL57`
