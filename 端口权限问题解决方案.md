# 端口权限问题解决方案

## 错误信息
```
Error: listen EACCES: permission denied 0.0.0.0:8080
```

## 问题原因

这个错误通常由以下原因引起：
1. **端口被占用** - 8080 端口已被其他进程使用
2. **权限不足** - 某些系统配置可能限制端口绑定
3. **防火墙/SELinux** - 系统安全策略阻止端口绑定
4. **端口范围限制** - 某些系统对非特权用户限制端口范围

## 解决方案

### 方案 1: 检查端口占用（推荐）

#### Linux/macOS
```bash
# 检查 8080 端口是否被占用
sudo lsof -i :8080
# 或
sudo netstat -tulpn | grep 8080
# 或
sudo ss -tulpn | grep 8080
```

#### Windows
```bash
# 检查 8080 端口是否被占用
netstat -ano | findstr :8080
```

如果端口被占用，可以：
- 停止占用端口的进程
- 或使用其他端口（见方案 2）

### 方案 2: 使用其他端口（最简单）

修改启动命令，使用其他端口（如 3000）：

```bash
# 使用环境变量指定端口
PORT=3000 node server.js

# 或 Windows PowerShell
$env:PORT=3000; node server.js

# 或 Windows CMD
set PORT=3000 && node server.js
```

### 方案 3: 检查并停止占用进程

#### Linux/macOS
```bash
# 查找占用 8080 端口的进程
sudo lsof -i :8080

# 停止进程（替换 PID 为实际进程ID）
sudo kill -9 <PID>
```

#### Windows
```bash
# 查找占用 8080 端口的进程
netstat -ano | findstr :8080

# 停止进程（替换 PID 为实际进程ID）
taskkill /PID <PID> /F
```

### 方案 4: 使用 sudo（不推荐，仅用于测试）

```bash
# 使用 root 权限运行（不推荐用于生产环境）
sudo node server.js
```

**⚠️ 注意：** 使用 sudo 运行 Node.js 应用存在安全风险，不推荐用于生产环境。

### 方案 5: 修改默认端口

如果 8080 端口经常被占用，可以修改 `server.js` 的默认端口：

```javascript
// 修改 server.js
const PORT = process.env.PORT || 3000; // 改为 3000 或其他端口
```

### 方案 6: 检查防火墙/SELinux（Linux）

#### 检查防火墙
```bash
# Ubuntu/Debian
sudo ufw status
sudo ufw allow 8080/tcp

# CentOS/RHEL
sudo firewall-cmd --list-ports
sudo firewall-cmd --add-port=8080/tcp --permanent
sudo firewall-cmd --reload
```

#### 检查 SELinux（如果启用）
```bash
# 检查 SELinux 状态
getenforce

# 如果启用，可以临时禁用测试（不推荐用于生产）
sudo setenforce 0
```

## 快速排查步骤

### 步骤 1: 检查端口占用
```bash
# Linux/macOS
sudo lsof -i :8080

# Windows
netstat -ano | findstr :8080
```

### 步骤 2: 如果端口被占用
- 停止占用进程，或
- 使用其他端口：`PORT=3000 node server.js`

### 步骤 3: 如果端口未被占用
- 尝试使用其他端口：`PORT=3000 node server.js`
- 检查防火墙设置
- 检查系统权限

### 步骤 4: 测试其他端口
```bash
# 使用 3000 端口测试
PORT=3000 node server.js

# 如果成功，说明是 8080 端口的问题
```

## 推荐解决方案

### 最佳实践：使用环境变量指定端口

1. **修改启动方式**（不修改代码）：
   ```bash
   PORT=3000 node server.js
   ```

2. **或修改默认端口**（修改代码）：
   ```javascript
   // server.js
   const PORT = process.env.PORT || 3000; // 改为 3000
   ```

3. **创建启动脚本**：
   ```bash
   # start.sh (Linux/macOS)
   #!/bin/bash
   PORT=3000 node server.js
   
   # start.bat (Windows)
   @echo off
   set PORT=3000
   node server.js
   ```

## 常见端口选择

- **3000** - 常用开发端口，通常不需要特殊权限
- **8000** - 常用替代端口
- **9000** - 常用替代端口
- **5000** - 常用替代端口

## 验证端口是否可用

```bash
# Linux/macOS
nc -zv localhost 3000

# Windows
Test-NetConnection -ComputerName localhost -Port 3000
```

## 总结

**最快解决方案：**
```bash
# 使用 3000 端口启动
PORT=3000 node server.js
```

**如果 3000 端口也被占用，尝试其他端口：**
```bash
PORT=8000 node server.js
PORT=9000 node server.js
```

**访问地址：**
- 本地：`http://localhost:3000`
- 局域网：`http://[服务器IP]:3000`
