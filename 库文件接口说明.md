# 库文件接口说明

## 接口需求

为了接入库文件选择功能，需要提供一个获取库文件列表的 API 接口。

## 建议的接口方式

### 方式一：RESTful API（推荐）

**接口地址：** `GET /api/library/images`

**请求参数（可选）：**
- `page`: 页码（默认：1）
- `pageSize`: 每页数量（默认：20）
- `keyword`: 搜索关键词（可选）
- `category`: 分类筛选（可选）

**响应格式：**
```json
{
  "code": 0,
  "message": "success",
  "data": {
    "images": [
      {
        "id": "image_001",
        "url": "https://example.com/images/image1.jpg",
        "thumbnail": "https://example.com/thumbnails/image1_thumb.jpg",
        "name": "图片名称1"
      },
      {
        "id": "image_002",
        "url": "https://example.com/images/image2.jpg",
        "thumbnail": "https://example.com/thumbnails/image2_thumb.jpg",
        "name": "图片名称2"
      }
    ],
    "total": 100,
    "page": 1,
    "pageSize": 20
  }
}
```

**字段说明：**
- `id`: 图片唯一标识符（必需）
- `url`: 图片的完整 URL，用于加载到节点中（必需）
- `thumbnail`: 缩略图 URL，用于在模态框中预览（可选，如果没有则使用 `url`）
- `name`: 图片名称，显示在预览下方（可选）

### 方式二：回调函数方式

如果无法提供 RESTful API，也可以使用回调函数方式：

**在 `NodeInspector.tsx` 中修改：**
```typescript
// 将 loadLibraryImages 函数改为接收回调函数
const loadLibraryImages = async (callback: (images: Array<{id: string, url: string, thumbnail?: string, name?: string}>) => void) => {
  // 调用外部提供的回调函数获取图片列表
  // 例如：window.libraryImageProvider?.getImages(callback)
};
```

## 当前实现

当前代码中，`loadLibraryImages` 函数会尝试调用 `/api/library/images` 接口。

**如果接口尚未接入：**
- 会显示 "暂无库文件" 的提示
- 不会报错，功能可以正常使用（只是库文件列表为空）

**接口接入后：**
- 只需要确保 `/api/library/images` 接口返回上述格式的数据
- 代码会自动加载并显示库文件
- 用户点击图片后，会自动填充到首帧图或尾帧图

## 测试建议

1. 可以先提供一个返回空数组的接口，确保模态框能正常打开
2. 然后逐步添加图片数据，测试显示和选择功能
3. 最后添加搜索、分页等高级功能

## 注意事项

1. **图片 URL 必须是可公开访问的**，因为需要在前端直接加载
2. **建议提供缩略图**，以提高加载速度和用户体验
3. **如果图片需要认证**，请确保 URL 包含必要的 token 或签名
4. **建议支持 CORS**，避免跨域问题
